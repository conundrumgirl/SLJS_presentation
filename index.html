History

Trivia: JavaScript name is trademarked (by Oracle, inheritedfrom Sun). That name comes from the standards organization Ecma

ECMAScript Evolution

  • 1995: JavaScript is born as LiveScript
  • 1997: ECMAScript standard is established - ECMAScript1
  • 1998: ECMAScript2 - minor changes
  • 1999: ES3 and IE5: regex, try /catch, new control statements etc.
  • 2000–2005: XMLHttpRequest, a.k.a. AJAX, Gmail (2004), Google Maps (2005).
  • 2008 - ECMAScript 4 (abandoned)
  • 2009: ES5 (this is what most of us use now)some new features eg. "strict mode” support for JSON
  • ECMAScript 6 (ES6) - became a standard on 17 June 2015
    compatibility table: http://kangax.github.io/compat-table/es6/

Let and Const

  • let more intuitive scoping and works nicely within for-loops.
  • const increases readability and decreases bugs
for(var i = 0; i < 10; i++) {
  foo[i].onclick = function() {
    console.log(i);
  };
}

for(let i = 0; i < 10; i++) {
  foo[i].onclick = function() {
    console.log(i);
  };
}

Gotchas:

  • Temporal Dead Zone: let/const declarations hoist, but throw errors when accessed before initialized more info
  • const is not “constant value” it is a constant reference to a value

Destructuring

A succinct way to extract values from objects and arrays.

Convenience

//objects
var foo = { bar: 'pony', baz: 3 }

var {bar, baz} = foo
var {bar: a, baz: b} = foo

//arrays
var [first, second, third] = someArray;

var [,,third] = ["foo", "bar", "baz"];
console.log(third);

//you can capture all trailing items in an array with a “rest” pattern:
var [head, ...tail] = [1, 2, 3, 4];
console.log(tail) // [2, 3, 4]
Method that returns an object 
function getCoords () {
  return {
    x: 10,
    y: 22
  }
}
var {x, y} = getCoords()
console.log(x) // <- 10

Swapping variables without the need for the aux variable 

function es5 () {
  var left = 10
  var right = 20
  var aux
  if (right > left) {
    aux = right
    right = left
    left = aux
  }
}

function es6 () {
  var left = 10
  var right = 20
  if (right > left) {
    [left, right] = [right, left]
  }
}

Promises

  • ability to write asynchronous code in a synchronous fashion, with flat indentation and a single exception channel.
  • help us unify asynchronous APIs and allow us to wrap non-spec compliant Promise APIs or callback APIs with real Promises.
  • give us guarantees of no race conditions and immutability of the future value represented by the Promise (unlike callbacks and events).
Create Promise 
var mypromise = new Promise(function(resolve, reject){
 // asynchronous code to run here
 // call resolve() to indicate task successfully completed
 // call reject() to indicate task has failed 
})

function getImage(url){
    return new Promise(function(resolve, reject){
        var img = new Image()
        img.onload = function(){
            resolve(url)
        }
        img.onerror = function(){
            reject(url)
        }
        img.src = url
    })
}
